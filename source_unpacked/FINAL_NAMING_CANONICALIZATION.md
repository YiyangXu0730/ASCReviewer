# ASCReview Naming Canonicalization

Canonical project name: `ASCReview`.

Historical aliases preserved for traceability:

- `The Final ACS Review`
- `THE FINAL ACS REVIEW.zip`
- User typo variants such as `AVS Review`

Final delivery rule:

- New final-delivery artifacts use `ASCReview` or `ASCREVIEW` in filenames.
- Historical `ACS` artifacts are archived, not deleted.
- `ACS` is not used as the canonical project name, package name, or canonical ZIP filename.

Reason:

The build package, prompts, skills, schemas, reports, and validation scripts consistently define the project as `ASCReview`. Delivery-layer aliases must not create a second project identity.
